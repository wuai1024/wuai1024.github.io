# 💻 Mac 上使用串口设备教程

本文介绍如何在 macOS 上使用串口设备（如 USB 转 TTL 模块、开发板串口）进行连接与调试。
全程使用系统自带工具和常见开源工具，无需额外图形界面。

# 🧩 一、检查是否识别到串口设备

  1. 将 USB 转串口设备插入 Mac。

  2. 打开终端（Terminal），执行：

    ls /dev/tty.*
    1

  3. 你会看到类似的输出：

    /dev/tty.usbserial-1420
/dev/tty.usbmodem14101
    1
    2

    一般来说：

    • /dev/tty.usbserial-xxxx → 常见于 CH340、CP2102、FT232 等芯片。
    • /dev/tty.usbmodemxxxx → 常见于 Arduino、STM32、ESP32 等开发板。

# ⚙️ 二、安装串口驱动（如有需要）

macOS 一般自带驱动，但某些芯片需手动安装：

芯片类型 驱动下载地址
CH340 / CH341 https://github.com/adrianmihalko/ch340g-ch34g-ch34x-mac-os-x-driver
CP210x https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
FTDI https://ftdichip.com/drivers/vcp-drivers/

安装后，重新插拔设备，再次运行 ls /dev/tty.* 确认设备出现。

# 🧰 三、使用系统自带命令连接串口

macOS 自带了一个简易串口工具 —— screen。

# 1. 查看串口名称

ls /dev/tty.*
1

假设你的设备是 /dev/tty.usbserial-1420。

# 2. 连接串口

screen /dev/tty.usbserial-1420 115200
1

其中 115200 为波特率，可根据设备设置调整，如 9600, 57600, 115200 等。

# 3. 退出 screen

按下以下组合键退出：

Ctrl + A  →  K  →  Y
1

# 🧪 四、使用 minicom 进行高级串口调试（推荐）

# 1. 安装 minicom

brew install minicom
1

# 2. 查看串口设备

ls /dev/tty.*
1

# 3. 配置串口

sudo minicom -s
1

进入配置界面后：

  • 选择 Serial port setup
  • 修改：
    A - Serial Device : /dev/tty.usbserial-1420
E - Bps/Par/Bits : 115200 8N1
    1
    2
  • 选择 Save setup as dfl 保存为默认配置

# 4. 启动串口

sudo minicom
1

# 5. 退出 minicom

按下：

Ctrl + A  →  Z  →  X
1

# 🧾 五、常见问题

问题 解决方案
没有 /dev/tty.usb* 设备 检查驱动是否正确安装，换 USB 接口或数据线
权限不足无法访问 执行 sudo chmod 666 /dev/tty.usb* 或以 sudo 启动程序
串口乱码 检查波特率是否匹配
无法输入命令 某些终端需打开本地回显（minicom 可设置 Local Echo）

# ✅ 七、快速使用总结

# 查看串口设备
ls /dev/tty.*

# 使用 screen 快速连接
screen /dev/tty.usbserial-1420 115200

# 或使用 minicom
brew install minicom
sudo minicom -D /dev/tty.usbserial-1420 -b 115200
