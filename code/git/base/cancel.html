      # 撤销工作区的更改（未提交到暂存区）

      git checkout -- <file>
      1

      # 撤销暂存区的更改

      git reset HEAD <file>
      1

      # 重置本地仓库到某个提交

      git reset <commit_sha>
      1

      # 撤销 git add 命令

      在 Git 中，如果你想撤销 git add 命令添加的文件，可以使用 git reset 命令。git reset 可以撤销已经暂存（staged）的文件，使其回到未暂存（unstaged）状态。具体步骤如下：

      # 撤销所有已添加的文件

      git reset
      1

      这会撤销所有已经添加到暂存区的文件，使它们回到未暂存状态。

      # 撤销特定文件

      如果你只想撤销特定文件，可以指定文件名：

      git reset <file>
      1

      例如：

      git reset example.txt
      1

      这样，example.txt 文件将从暂存区中移除。

      # 查看已添加的文件

      你可以使用以下命令查看当前暂存的文件列表：

      git status
      1

      这将显示已添加但未提交的文件列表，以及未暂存的文件。

      # 撤销目录下的所有文件

      如果想撤销某个目录下的所有文件，可以指定目录路径：

      git reset <directory>/
      1

      例如：

      git reset .idea/
      1

      这样，src 目录下的所有文件将从暂存区中移除。
